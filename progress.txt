## Codebase Patterns
- Documentação de planejamento vai em arquivos markdown na raiz do projeto
- PRD em formato JSON na raiz do projeto (prd.json)
- Branch de trabalho definida no PRD via campo branchName
- Docs técnicos ficam em /docs/ (business-rules.md, architecture.md)
- Sistema suporta XV e Sevens — NUNCA hardcodar parâmetros de tipo de jogo
- Dois perfis: Gestor (admin) e Quarto Árbitro (operador)
- Offline-first: toda lógica no browser, servidor é backup
- Placar SEMPRE derivado da soma de eventos, nunca armazenado
- Cronômetros disciplinares pausam com jogo, médicos NÃO
- JWT com refresh token longo para auth offline
- UUIDs gerados no client para operação offline
- Cadastros mestres são readonly offline, apenas dados de partida são "offline wins"
- App fica em /app (Next.js project). Rodar `cd app && pnpm dev` para iniciar.
- shadcn/ui usa base-ui (não Radix). Select.onValueChange recebe `string | null`, precisa `v ?? ""`
- Não usar `asChild` — não existe no base-ui. Usar state controlado para dialogs.
- Google Fonts não disponível em build offline — usar font-sans do sistema
